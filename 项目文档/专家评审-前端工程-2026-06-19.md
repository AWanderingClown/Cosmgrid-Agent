# 专家评审：Cosmgrid-Agent v0.2 前端架构（精简版）

> **评审对象**：v0.2 前端实施步骤（3 页面 + 7 组件 + shadcn/ui init + 流式对话 + crypto + RHF）
> **评审者**：前端工程专家
> **评审日期**：2026-06-19（2026-06-19 精简）
> **状态**：v0.2 前端**未实施**（任务 #6 留 v0.2 前端阶段）；本报告是实施指南

> **结论一句话**：**v0.2 前端 17 小时 ≈ 2 个工作日**（ChatPage 先做 → ProvidersPage → ModelsPage → 验收）

---

## 一、关键发现

1. **shadcn/ui init 在 React 19 + Tailwind v4 上**已稳定支持，命令是 `pnpm dlx shadcn@latest init -d`（**注意是 `shadcn@latest`，不是过时的 `shadcn-ui@latest`**）。Tailwind v4 不再需要 `tailwind.config.ts`，CSS 变量直接走 `@theme inline` 写在 `globals.css` 里——v0.2 必须用最新骨架。
2. **`@ai-sdk/react` 的 `useChat` hook + `DefaultChatTransport`** 是 2026 流式消费的官方解。手动 `fetch + ReadableStream` 只在写自定义 transport 时才用；v0.2 的 ChatPage 直接用 `useChat`，开发量砍 80%。
3. **react-router v6/v7 在 Tauri 桌面 App 里是过度设计**——3 页面 + 侧栏 + 无 URL 同步需求，用 `useState` 切换 Page 组件就够。路由 5 行代码，别上库。
4. **`useChat` 自带 `status: 'submitted' | 'streaming' | 'ready' | 'error'` 状态机**，前端 UI 加载态、禁用按钮、流式展示全部由此驱动——不需要自己写 `isLoading`。
5. **RHF + zodResolver 在 3 个页面完全够用**——AddProviderDialog 是密集表单（最复杂），ProvidersPage/ModelsPage 主要是列表 + 弹窗触发器。Zustand 在 v0.2 不需要。
6. **`extractUsage(value: any)` 的 `any` 必须改 `unknown` + type guard**——这是 v0.2 后端 `usage-extractor.ts` 的硬指标，`any` 会让 `noImplicitAny` 编译告警，且 TS 类型推理完全失效。

### 4.1 ❤️ 必须改：ChatPage 用 `useChat` 而不是手写 fetch

```tsx
// app/src/pages/ChatPage.tsx
import { useChat } from "@ai-sdk/react"
import { DefaultChatTransport } from "ai"
import { useMemo, useState } from "react"
import { Button } from "@/components/ui/button"
import { Input } from "@/components/ui/input"
import { Select } from "@/components/ui/select"
import { useModelsByWorkRole } from "@/hooks/useModelsByWorkRole"

export function ChatPage() {
  const [modelId, setModelId] = useState<string>("")
  const { data: models } = useModelsByWorkRole("main_chat")
  const activeModelId = modelId || models?.[0]?.id || ""

  const transport = useMemo(
    () =>
      new DefaultChatTransport({
        api: "/api/chat/stream",
        // 每条消息发请求时带上当前选中的模型 ID
        prepareSendMessagesRequest: ({ messages }) => ({
          body: { modelId: activeModelId, messages },
        }),
      }),
    [activeModelId],
  )

  const { messages, sendMessage, status, stop, error } = useChat({ transport })
  const isStreaming = status === "submitted" || status === "streaming"

  return (
    <div className="flex h-full flex-col gap-4 p-6">
      <div className="flex items-center gap-2">
        <Select value={activeModelId} onValueChange={setModelId}>
          {models?.map((m) => (
            <SelectItem key={m.id} value={m.id}>
              {m.displayName}
            </SelectItem>
          ))}
        </Select>
      </div>

      <div className="flex-1 overflow-y-auto space-y-3">
        {messages.map((m) => (
          <div
            key={m.id}
            className={m.role === "user" ? "text-right" : "text-left"}
          >
            {m.parts.map((part, i) =>
              part.type === "text" ? (
                <p key={i} className="rounded-lg bg-muted px-3 py-2 inline-block">
                  {part.text}
                </p>
              ) : null,
            )}
          </div>
        ))}
        {error && <p className="text-destructive text-sm">错误：{error.message}</p>}
      </div>

      <form
        onSubmit={(e) => {
          e.preventDefault()
          const text = new FormData(e.currentTarget).get("input") as string
          if (!text.trim()) return
          sendMessage({ text })
          e.currentTarget.reset()
        }}
        className="flex gap-2"
      >
        <Input name="input" placeholder="说点什么..." disabled={isStreaming} />
        {isStreaming ? (
          <Button type="button" onClick={() => stop()}>停止</Button>
        ) : (
          <Button type="submit" disabled={!activeModelId}>发送</Button>
        )}
      </form>
    </div>
  )
}
```

**为什么这个更好**：
- `useChat` 自带 `status` 状态机 → 不用 `useState({ loading })` 自己管
- `messages.parts` 类型安全 → 不解析原始 SSE
- `stop()` 一行代码 → 不手写 `AbortController`
- `DefaultChatTransport` 跟后端 `result.toUIMessageStreamResponse()` 配对 → 协议对齐零摩擦

### 4.3 ❤️ 必须改：`extractUsage` 用 `unknown` + type guard

```ts
// app/server/llm/usage-extractor.ts
interface Usage {
  inputTokens?: number
  outputTokens?: number
  cacheReadInputTokens?: number
  cacheWriteInputTokens?: number
  reasoningTokens?: number
  totalTokens?: number
}

// Type guard：先验证是对象，再逐字段读
function isUsageLike(raw: unknown): raw is Record<string, unknown> {
  return typeof raw === "object" && raw !== null
}

function pickNumber(raw: Record<string, unknown>, ...keys: string[]): number | undefined {
  for (const k of keys) {
    const v = raw[k]
    if (typeof v === "number" && Number.isFinite(v)) return v
  }
  return undefined
}

export function extractUsage(raw: unknown): Usage {
  if (!isUsageLike(raw)) return {}
  const cachedTokens = raw["inputTokenDetails"]
  const cachedRead =
    (isUsageLike(cachedTokens) && typeof cachedTokens["cacheReadTokens"] === "number"
      ? cachedTokens["cacheReadTokens"]
      : undefined)
  return {
    inputTokens: pickNumber(raw, "inputTokens", "promptTokens"),
    outputTokens: pickNumber(raw, "outputTokens", "completionTokens"),
    cacheReadInputTokens: pickNumber(raw, "cacheReadInputTokens", "cachedTokens") ?? cachedRead,
    cacheWriteInputTokens: pickNumber(raw, "cacheWriteInputTokens"),
    reasoningTokens: pickNumber(raw, "reasoningTokens"),
    totalTokens: pickNumber(raw, "totalTokens"),
  }
}
```

**为什么改**：
- `any` 在 `noImplicitAny` 严格模式下编译告警
- `unknown` + type guard 强制每个字段都做类型检查，不会把 `null` 当 number 算
- `isUsageLike` 让 TS 知道分支后是 `Record<string, unknown>`，可继续 `.[]` 访问
- 测试覆盖更容易（传 `null` / `undefined` / 字符串 / 对象 都返回 `{}`）

### 4.5 💚 直接做（参考）：AddProviderDialog 用 RHF + zodResolver

```tsx
// app/src/components/providers/AddProviderDialog.tsx
import { useForm } from "react-hook-form"
import { zodResolver } from "@hookform/resolvers/zod"
import { Dialog, DialogContent, DialogHeader, DialogTitle } from "@/components/ui/dialog"
import { Form, FormField, FormItem, FormLabel, FormControl, FormMessage } from "@/components/ui/form"
import { Input } from "@/components/ui/input"
import { Button } from "@/components/ui/button"
import { providerFormSchema, type ProviderFormValues } from "@/lib/schemas/provider"
import { encryptApiKey } from "@/lib/crypto"
import { BasicFormFields } from "./BasicFormFields"
import { ApiKeyInput } from "./ApiKeyInput"
import { ProviderTypeSelect } from "./ProviderTypeSelect"
import { WorkRoleSelector } from "./WorkRoleSelector"
import { ModelConfigFields } from "./ModelConfigFields"
import { TestConnectionButton } from "./TestConnectionButton"
import { useCreateProvider } from "@/hooks/useCreateProvider"

interface Props {
  open: boolean
  onOpenChange: (open: boolean) => void
}

export function AddProviderDialog({ open, onOpenChange }: Props) {
  const form = useForm<ProviderFormValues>({
    resolver: zodResolver(providerFormSchema),
    defaultValues: {
      enabled: true,
      supportsStreaming: true,
      supportsFunctionCall: true,
      supportsVision: false,
      workRoles: ["main_chat"],
      capabilityTags: [],
      capabilityScore: {},
    },
  })

  const createProvider = useCreateProvider()

  const onSubmit = async (values: ProviderFormValues) => {
    try {
      const apiKeyEncrypted = await encryptApiKey(values.apiKey)
      await createProvider.mutateAsync({ ...values, apiKeyEncrypted })
      onOpenChange(false)
      form.reset()
    } catch (err) {
      form.setError("root", {
        message: err instanceof Error ? err.message : "添加失败",
      })
    }
  }

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent className="max-w-2xl max-h-[90vh] overflow-y-auto">
        <DialogHeader>
          <DialogTitle>添加 API 接入</DialogTitle>
        </DialogHeader>
        <Form {...form}>
          <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-6">
            <BasicFormFields form={form} />
            <ProviderTypeSelect form={form} />
            <ApiKeyInput form={form} />
            <TestConnectionButton form={form} />
            <ModelConfigFields form={form} />
            <WorkRoleSelector
              value={form.watch("workRoles")}
              onChange={(roles) => form.setValue("workRoles", roles, { shouldValidate: true })}
            />
            <div className="flex justify-end gap-2">
              <Button type="button" variant="outline" onClick={() => onOpenChange(false)}>
                取消
              </Button>
              <Button type="submit" disabled={createProvider.isPending}>
                {createProvider.isPending ? "保存中..." : "保存"}
              </Button>
            </div>
          </form>
        </Form>
      </DialogContent>
    </Dialog>
  )
}
```

**说明**：每个字段子组件接收 `form: UseFormReturn<ProviderFormValues>`，内部用 `<FormField control={form.control} name="..." />` 包 Input/Label/ErrorMessage，符合 CC Switch RHF 用法。

### 4.6 💚 直接做：`src/lib/api.ts` fetch wrapper

```ts
// app/src/lib/api.ts
export class ApiError extends Error {
  constructor(public status: number, message: string, public detail?: unknown) {
    super(message)
    this.name = "ApiError"
  }
}

async function request<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(path, {
    headers: { "Content-Type": "application/json", ...init?.headers },
    ...init,
  })
  if (!res.ok) {
    const body = await res.json().catch(() => ({}))
    throw new ApiError(res.status, body.error ?? res.statusText, body)
  }
  return res.json() as Promise<T>
}

export const api = {
  get: <T>(path: string) => request<T>(path),
  post: <T>(path: string, body: unknown) =>
    request<T>(path, { method: "POST", body: JSON.stringify(body) }),
  patch: <T>(path: string, body: unknown) =>
    request<T>(path, { method: "PATCH", body: JSON.stringify(body) }),
  delete: <T>(path: string) => request<T>(path, { method: "DELETE" }),
}
```

**说明**：原生 fetch + 一个 ApiError 类，**不引 axios / ky / swr**。30 行代码搞定；React Query 也不引（v0.2 用 `useMutation` / `useQuery` 自己写，50 行以内）。

## 七、最终推荐

> **v0.2 前端架构：**
> - **路由**：`useState<PageKey>`（**不要 react-router**）
> - **流式**：`@ai-sdk/react` 的 `useChat` + `DefaultChatTransport`（**不要手写 fetch + ReadableStream**）

> - **workRoles**：Popover + Command（cmdk）搜索 + Checkbox（**不要 4x2 grid**）

> - **shadcn**：用 `pnpm dlx shadcn@latest init -d`（**不是过时的 `shadcn-ui@latest`**）
