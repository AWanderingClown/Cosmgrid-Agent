## 一、关键发现（先看这段，5 条最重要的）

### 发现 1：Vercel AI SDK **6.0.208 是当前最新**（2026-06），v0.2 选 6.x 没问题，但要用新 transport API

- 官方包名仍是 `ai`（不是 `vercel-ai-sdk`），最新版本 **6.0.208**（npm registry 直查）
- AI SDK 5 早已发布、6 已稳定，v0.2 跳到 6 是对的
- **重大 API 变化**：AI SDK 6 把 transport 抽成了一等公民。前端用 `useChat` + `DefaultChatTransport({ api: '/api/chat' })`，**不再强依赖 Next.js**——可以指向任意 HTTP 后端（包括 Hono）
- **`@ai-sdk/react@3.0.210`** 同时支持 React 18 和 19，**v0.2 用 React 19 没问题**
- 关键参考：https://ai-sdk.dev/docs/ai-sdk-ui/transport

### 发现 2：**不要用 Hono `streamSSE()` 包 AI SDK 流**——直接 `return result.toUIMessageStreamResponse()`

- Vercel 官方示例（Hono + streamText）就一行：`return result.toUIMessageStreamResponse()`
- AI SDK 自带的 UI message stream 协议已经处理了 tool call / reasoning / sources / files 等所有 parts
- 走 `streamSSE()` 自己拼 `{ type: "text", content: chunk }` 是 OpenCode 风格（v0.2 报告已抄），**但这是 OpenCode 自己的协议，不是 Vercel 的**——后续接 tool use 会全推倒重写
- 关键参考：https://ai-sdk.dev/docs/ai-sdk-ui/chatbot

### 发现 3：**shadcn 包名变了** —— 用 `npx shadcn@latest init`，不是 `npx shadcn-ui@latest init`

- 2026 年的官方包名是 **`shadcn`**，GitHub 组织仍是 `shadcn-ui`（仓库地址没变）
- v0.2 报告里所有 `npx shadcn-ui@latest` 命令都要改成 `npx shadcn@latest`
- shadcn 现已**完全支持 Tailwind v4 + React 19**，新项目默认就是这套
- Tailwind v4.3 已是稳定版（CSS-first config，`@import "tailwindcss"`，无 `tailwind.config.js`）
- 关键参考：https://ui.shadcn.com/docs/installation

### 发现 4：**Tauri 2 用 IPC（invoke）而不是 HTTP**——但 v0.2 用 Hono HTTP 服务也合理，需明确边界

- Tauri 官方推荐 **优先用 IPC 命令**（`invoke`），不要开本地 HTTP 服务器（攻击面更大、binary 更复杂）
- 但 v0.2 必须用 HTTP 走 Hono，原因：
  1. **流式 SSE 必须 HTTP**——Tauri IPC 不支持长连接流
  2. **AI SDK 的 `useChat` 默认就是 HTTP transport**——改 IPC 等于全重写
  3. **v0.1 已经定了 Hono + Hono Node Server**，不能改
- **正确边界**：CRUD / 状态读取 → 走 Hono HTTP（前端 `<-> Hono`）；如果以后做"重型本地操作"（文件批处理等）→ 走 Tauri IPC（前端 `<-> Rust`）。v0.2 全 HTTP 没问题
- 关键参考：https://v2.tauri.app/concept/inter-process-communication/

### 发现 5：**API Key 加密方案** —— 用 `tauri-plugin-keyring`（OS 钥匙串），不要用固定密钥

- v0.2 报告里推荐"固定密钥派生"（`"cosmgrid-agent-v0.2-fixed-passphrase"`），这是 **C 简化版**——**真要上线会有合规问题**
- **正解**：用 **`tauri-plugin-keyring`**，背后是 OS 原生：
  - macOS → Keychain
  - Windows → Credential Manager
  - Linux → Secret Service
- 不用 Stronghold（v0.2 报告已正确排除）：Stronghold 适合存 vault/多 key，API key 这种**单 key 走 OS 钥匙串**更轻
- v0.2 简化策略：**keyring 里只存 master password**，用 master password 派生 AES-GCM key 来加密 SQLite 里的 apiKeyEncrypted 字段。比固定密钥强 100 倍，比全 keyring 灵活
- 关键参考：https://v2.tauri.app/plugin/stronghold/（强但不必要）、`tauri-plugin-keyring` 文档

## 三、技术替代建议（针对 💛 和 ❤️ 的具体方案）

### ❤️ 替代 1：`chat-handler.ts` 改用 `result.toUIMessageStreamResponse()`

**当前方案**（v0.2 报告）：
```ts
c.header("Content-Type", "text/event-stream")
return streamSSE(c, async (stream) => {
  for await (const chunk of result.textStream) {
    await stream.writeSSE({ data: JSON.stringify({ type: "text", content: chunk }) })
  }
})
```

**新方案**（Vercel AI SDK 6 官方）：
```ts
// app/server/routes/chat.ts
import { streamText, convertToModelMessages, type UIMessage } from "ai"

app.post("/api/chat", async (c) => {
  const { messages, modelId }: { messages: UIMessage[]; modelId: string } = await c.req.json()

  // 查 DB 取 provider + api key + model
  const { model, apiKey } = await loadModelConfig(modelId)

  const result = streamText({
    model: getLanguageModel(model.provider.type, model.name, apiKey),
    messages: await convertToModelMessages(messages),
    onFinish: async ({ usage }) => {
      await prisma.usageEvent.create({ data: { /* ... */ } })
    },
  })

  // 一行返回，自动设置 SSE headers + Content-Type
  return result.toUIMessageStreamResponse()
})
```

**收益**：
- 从 15 行变 1 行
- 原生支持 tool call / reasoning / sources / files（v0.3 不用重写）
- 前端用 `useChat` 直接吃这个协议

**参考**：https://ai-sdk.dev/docs/ai-sdk-ui/chatbot（官方 Hono 示例）

### ❤️ 替代 2：`ChatPage.tsx` 改用 `useChat` + `DefaultChatTransport`

**当前方案**（v0.2 报告）：手写 fetch + ReadableStream + 自己解析 SSE + 自己维护 messages state

**新方案**（AI SDK 6 官方）：
```tsx
// app/src/pages/ChatPage.tsx
import { useChat } from "@ai-sdk/react"
import { DefaultChatTransport } from "ai"

export function ChatPage() {
  const { messages, sendMessage, status, error, stop } = useChat({
    transport: new DefaultChatTransport({
      api: "/api/chat",
      // Hono 端点，跟 Next.js 没区别
      prepareSendMessagesRequest: ({ messages }) => ({
        body: { messages, modelId: selectedModelId },
      }),
    }),
  })

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.parts.map(part => {
            if (part.type === "text") return <p>{part.text}</p>
            // future: tool invocations / reasoning / sources 自动处理
          })}
        </div>
      ))}
      <form onSubmit={e => {
        e.preventDefault()
        const input = e.currentTarget.elements.namedItem("message") as HTMLInputElement
        sendMessage({ text: input.value })
        input.value = ""
      }}>
        <input name="message" placeholder="输入消息..." disabled={status !== "ready"} />
        <button type="submit">发送</button>
        {status === "streaming" && <button onClick={() => stop()}>停止</button>}
      </form>
    </div>
  )
}
```

**收益**：
- 从 ~300 行手写 fetch/parse/state 变 ~30 行
- **自动处理**：流式追加、错误重连、abort、regenerate、tool calls、reasoning、sources、files
- `status` 自动暴露 `submitted` / `streaming` / `ready` / `error`
- 后续 v0.3 加 tool use（让 AI 调本地工具）零代码改动

**参考**：https://ai-sdk.dev/docs/ai-sdk-ui/transport

### ❤️ 替代 3：`crypto.ts` 改用 `tauri-plugin-keyring`

**当前方案**（v0.2 报告）：固定密钥派生，源码里写死 passphrase

**新方案**：keyring 存 master password + AES-GCM 加密
```ts
// app/src-tauri/src/commands/crypto.rs（新文件）
use tauri_plugin_keyring::KeyringExt;

#[tauri::command]
async fn get_master_key(app: tauri::AppHandle) -> Result<String, String> {
  // 从 OS Keychain 读 master password；首次启动随机生成并存入
  let entry = app.keyring().get_entry("cosmgrid-agent", "master")?;
  Ok(entry.get_password().unwrap_or_else(|_| {
    let new_pass = generate_random_password(); // 32 字节随机
    let _ = app.keyring().create_entry("cosmgrid-agent", "master")
      .and_then(|e| e.set_password(&new_pass));
    new_pass
  }))
}
```

```ts
// app/src/lib/crypto.ts（前端）
import { invoke } from "@tauri-apps/api/core"

const cachedKey: { key: CryptoKey } | null = null

async function getKey(): Promise<CryptoKey> {
  if (cachedKey) return cachedKey.key
  const passphrase = await invoke<string>("get_master_key")
  const keyMaterial = await crypto.subtle.importKey(
    "raw", new TextEncoder().encode(passphrase), "PBKDF2", false, ["deriveKey"]
  )
  const key = await crypto.subtle.deriveKey(
    { name: "PBKDF2", salt: new TextEncoder().encode("cosmgrid-agent-v0.2"), iterations: 100_000, hash: "SHA-256" },
    keyMaterial,
    { name: "AES-GCM", length: 256 }, false, ["encrypt", "decrypt"]
  )
  cachedKey = { key }
  return key
}

// encryptApiKey / decryptApiKey 同 v0.2 报告
```

**收益**：
- master password 32 字节随机，OS Keychain 加密存储，逆向攻击者要破 2 层（Keychain + AES-GCM）
- 符合企业合规要求
- 用户换电脑 / 重装系统 → Keychain 自动同步（macOS）或要重新授权（其他平台），符合预期

**依赖**：
```bash
# Rust 侧
cargo add tauri-plugin-keyring
# app/src-tauri/src/lib.rs 注册
.plugin(tauri_plugin_keyring::init())
```

**参考**：https://github.com/tauri-apps/plugins-workspace/tree/v2/plugins/keyring

## 四、风险与坑

### 风险 1：AI SDK 6 仍处于 6.0.x patch 阶段，可能有 breaking changes
- 6.0.208 是 patch 208，说明 API 已经稳定但仍在修 bug
- **对策**：锁版本 `"ai": "^6.0.208"` + `"@ai-sdk/react": "^3.0.210"` + `"@ai-sdk/anthropic": "^3.x"` + `"@ai-sdk/openai": "^3.x"` + `"@ai-sdk/google": "^3.x"`
- npm 上各 provider 包都是 3.x，跟 core 包对齐
- 监控：每 2 周跑一次 `pnpm outdated`，遇到 minor 升级手动 review

### 风险 2：shadcn Tailwind v4 + React 19 在 Tauri webview 上可能不稳
- Tauri 用系统 WebView（macOS = WKWebView），某些 CSS 新特性在 Safari 17 才支持
- **对策**：v0.2 初始化后立刻 `pnpm tauri dev` 跑一遍，看页面渲染、字体、动画对不对

### 风险 3：`tauri-plugin-keyring` 在 Linux 上行为不一致
- macOS / Windows 用 OS 原生 API
- Linux 用 Secret Service（gnome-keyring / KWallet），headless 服务器没装会报错
- **对策**：v0.2 只承诺 macOS / Windows 支持（Cosmgrid-Agent 用户都是桌面），Linux 留 v0.3

### 风险 4：`useChat` 的状态管理跟 React Query/Zustand 冲突
- `useChat` 内部维护 `messages` state，外部拿不到
- v0.2 不需要交叉读 messages（不重），所以问题不大
- **对策**：v0.2 不要尝试把 `useChat` 的 messages 同步到 Zustand；如真要，v0.3 看 AI SDK 是否有 `externalMessages` API

### 风险 5：`tauri-plugin-keyring` 增加 Rust 依赖，编译时间会涨
- keyring crate 本身不大，但它的 deps 链（secret-service / security-framework / winapi）不小
- **对策**：cold build 时间从 ~30s 涨到 ~1min，CI 上提前缓存 `target/` 即可
