### 3.1 fetch + ApiResponse<T> 拆包样板 × 4 处

| 文件:行号 | 模式 |
|----------|------|
| `ChatPage.tsx:62-67` | `fetch().then(r => r.json() as Promise<ModelsResponse>)` + `if (!res.success) throw` |
| `ChatPage.tsx:106-127` | `fetch().then(r => r.json() as Promise<StreamResponse>)` + `if (b.success) ...` |
| `ProvidersPage.tsx:49-57` | `Promise.all([fetch...]).then(([p,c,m]) => { if (p.success) ... })` |
| `AddProviderDialog.tsx:84-138` | 4 次 `fetch().json() + if (!body.success) throw` |

**重构**：抽 `src/lib/api-client.ts`：

```ts
// src/lib/api-client.ts
interface ApiOk<T> { success: true; data: T }
interface ApiFail { success: false; error: string }
type ApiResponse<T> = ApiOk<T> | ApiFail

export async function apiGet<T>(url: string): Promise<T> {
  const r = await fetch(url)
  const body = (await r.json()) as ApiResponse<T>
  if (!body.success) throw new Error(body.error)
  return body.data
}

export async function apiPost<T>(url: string, body: unknown): Promise<T> {
  const r = await fetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  })
  const json = (await r.json()) as ApiResponse<T>
  if (!json.success) throw new Error(json.error)
  return json.data
}

export async function apiPatch<T>(url: string, body: unknown): Promise<T> { /* ... */ }
export async function apiDelete(url: string): Promise<void> { /* ... */ }
```

**收益**：
- AddProviderDialog 4 个 `fetch().json() + if (!body.success) throw` → 4 行 `apiPost<{id:string}>(...)` 调用
- ChatPage 加载逻辑从 11 行 → 3 行
- ProvidersPage 的 `load()` 从 9 行 → 3 行
- 净减 **~40 行** + 统一错误处理路径
