---

**核心结论**：v0.2 处理的是用户最敏感的东西——**API Key（真金白银）**。当前草案（CC Switch 模式明文 + 固定密钥派生）**不够安全**，但对桌面应用来说**可接受**（因为 Tauri 沙箱已经隔离了大部分远程攻击者）。建议：

| 项 | 草案评级 | 处理方式 |
|---|---|---|
| **加密方案**（固定密钥派生） | ❤️ 必须改 | 改用 **OS keychain 优先**，**PBKDF2+OS 设备指纹 fallback** |
| **CORS 配置** | 💚 安全 OK | 当前 3 个 origin 合理，加 `tauri://localhost` 已 OK |
| **测试连接限流** | 💛 加固后用 | 加客户端 debounce 2s + 服务端 per-IP 5 次/分钟 |
| **错误信息泄漏** | ❤️ 必须改 | API Key 前缀必须从 401/403 错误中剥离 |
| **UsageEvent** | 💚 安全 OK | 字段已隔离，**禁止扩展 schema 加 message/apiKey** |
| **Tauri IPC** | 💛 加固后用 | test_provider_connection command 要加超时 + 不写盘 |
| **审计日志** | 💛 加固后用 | v0.2 UsageEvent 足够，v0.4 加 SecurityEvent 表 |
| **CSP** | 💛 加固后用 | tauri.conf.json 当前 `csp: null`，**必须**加严格 CSP |

**最终推荐**：v0.2 安全方案 = **OS keychain（AES-GCM + macOS Keychain / Windows DPAPI / Linux Secret Service）+ 固定密钥派生 fallback** + **客户端 debounce 2s** + **服务端 per-IP 限流** + **错误清洗（API Key 前缀剥离）** + **Tauri 严格 CSP**。

## 二、关键风险（按严重度排序）

### 🔴 P0-1：API Key 固定密钥派生 = 源码泄漏即破产

**严重度**：CRITICAL（如果有人拿到源码）
**风险描述**：
- `crypto.ts:9` 把 PASSPHRASE `"cosmgrid-agent-v0.2-fixed-passphrase"` 硬编码在源码里
- Salt `"cosmgrid-agent-v0.2-salt"` 也是固定字符串
- **Tauri app 打包后 JS bundle 在 app.asar 里可解包**（asar 是明文压缩，不是加密），逆向工程 5 分钟拿到密钥
- 一旦密钥泄漏，所有 Cosmgrid-Agent v0.2 用户的加密 API Key 全可解

**攻击场景**：
1. 攻击者下载 Cosmgrid-Agent.dmg（开源或破解后）
2. 用 `npx asar extract app.asar extracted/` 解包
3. `grep -r "PASSPHRASE" extracted/` 拿到密钥
4. 写 10 行 JS 批量解所有 v0.2 用户的 API Key
5. **批量盗刷**（Anthropic/OpenAI 余额）

**当前防护**：
- ❌ 无 OS keychain 集成
- ❌ 密钥在源码明文
- ⚠️ Tauri 沙箱隔离了**远程**攻击者（不能直接读用户磁盘），但**本地**有恶意软件/同机用户/物理访问时=完全暴露

**改进建议（必须改）**：

#### 方案 A：OS Keychain 优先 + 固定密钥派生 fallback（v0.2 推荐）

```typescript
// app/src/lib/crypto.ts — v0.2 改进版
// 策略：优先 OS keychain；fallback 用 PBKDF2 + 设备指纹派生（仍然不存磁盘）

import { invoke } from "@tauri-apps/api/core";

// 加密参数（仅 iv 需随机，key 来自 keychain 或 fallback）
const ALGORITHM = "AES-GCM";
const KEY_LENGTH = 256;
const PBKDF2_ITERATIONS = 600_000; // OWASP 2023 推荐（100k 已不达标）

export interface EncryptionKey {
  key: CryptoKey;
  source: "keychain" | "derived";
}

let cachedKey: EncryptionKey | null = null;
let cachedKeyPromise: Promise<EncryptionKey> | null = null;

/**
 * 获取加密密钥（单例 + Promise cache，避免并发重复获取）
 * 优先级：
 *   1. macOS Keychain / Windows DPAPI / Linux Secret Service（OS keychain）
 *   2. PBKDF2 从设备指纹派生（fallback，无持久存储）
 */
export async function getEncryptionKey(): Promise<EncryptionKey> {
  if (cachedKey) return cachedKey;
  if (cachedKeyPromise) return cachedKeyPromise;

  cachedKeyPromise = (async () => {
    // 1. 尝试从 OS keychain 取
    try {
      const rawKeyB64 = await invoke<string | null>("get_api_key_encryption_key");
      if (rawKeyB64) {
        const rawKey = Uint8Array.from(atob(rawKeyB64), c => c.charCodeAt(0));
        const key = await crypto.subtle.importKey(
          "raw", rawKey, ALGORITHM, false, ["encrypt", "decrypt"]
        );
        cachedKey = { key, source: "keychain" };
        return cachedKey;
      }
    } catch (err) {
      console.warn("[crypto] OS keychain 不可用，降级到派生方案", err);
    }

    // 2. fallback：PBKDF2 从设备指纹派生
    const deviceFingerprint = await getDeviceFingerprint();
    const salt = await getDeviceSalt(); // 同样从设备指纹 + 静态值派生

    const keyMaterial = await crypto.subtle.importKey(
      "raw",
      new TextEncoder().encode(deviceFingerprint),
      "PBKDF2",
      false,
      ["deriveKey"]
    );
    const key = await crypto.subtle.deriveKey(
      {
        name: "PBKDF2",
        salt,
        iterations: PBKDF2_ITERATIONS,
        hash: "SHA-256",
      },
      keyMaterial,
      { name: ALGORITHM, length: KEY_LENGTH },
      false,
      ["encrypt", "decrypt"]
    );
    cachedKey = { key, source: "derived" };
    return cachedKey;
  })();

  return cachedKeyPromise;
}

/**
 * 设备指纹：hostname + platform + arch + 一个静态盐
 * 注意：这不是真正的"用户身份"，仅用于让加密结果在不同设备上不通用
 * （即使 A 设备的源码泄漏，B 设备的密文也无法解）
 */
async function getDeviceFingerprint(): Promise<string> {
  const platform = await invoke<string>("get_platform_info"); // { os, arch, hostname }
  return `cosmgrid:${platform.os}:${platform.arch}:${platform.hostname}`;
}

async function getDeviceSalt(): Promise<Uint8Array> {
  // salt 也从设备指纹派生（确定性，但不同设备不同）
  const fp = await getDeviceFingerprint();
  const hash = await crypto.subtle.digest("SHA-256", new TextEncoder().encode(`salt:${fp}`));
  return new Uint8Array(hash);
}

/**
 * 加密 API Key → base64(iv || ciphertext)
 * 每次调用都用随机 iv（12 字节）
 */
export async function encryptApiKey(plain: string): Promise<string> {
  const { key } = await getEncryptionKey();
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const ciphertext = await crypto.subtle.encrypt(
    { name: ALGORITHM, iv },
    key,
    new TextEncoder().encode(plain)
  );
  const combined = new Uint8Array(iv.length + ciphertext.byteLength);
  combined.set(iv);
  combined.set(new Uint8Array(ciphertext), iv.length);
  return btoa(String.fromCharCode(...combined));
}

export async function decryptApiKey(b64: string): Promise<string> {
  const { key } = await getEncryptionKey();
  const combined = Uint8Array.from(atob(b64), c => c.charCodeAt(0));
  const iv = combined.slice(0, 12);
  const ciphertext = combined.slice(12);
  const plaintext = await crypto.subtle.decrypt(
    { name: ALGORITHM, iv },
    key,
    ciphertext
  );
  return new TextDecoder().decode(plaintext);
}

/**
 * 清空内存中的密钥（应用退出时调用，降低 heap dump 风险）
 */
export function clearCachedKey(): void {
  cachedKey = null;
  cachedKeyPromise = null;
}
```

#### Tauri Rust 端（`app/src-tauri/src/commands/keychain.rs`，新文件）

```rust
// macOS Keychain / Windows DPAPI / Linux Secret Service 集成
// 用 tauri-plugin-stronghold 或直接调 OS API

use serde::{Deserialize, Serialize};

const KEYCHAIN_SERVICE: &str = "com.cosmgrid.agent.api-key-encryption";
const KEYCHAIN_ACCOUNT: &str = "encryption-key-v1";

#[derive(Debug, Serialize, Deserialize)]
pub struct KeychainResult {
    pub found: bool,
    pub key_b64: Option<String>,
}

#[tauri::command]
pub async fn get_api_key_encryption_key() -> Result<KeychainResult, String> {
    // 用 keyring crate（跨平台）
    let entry = keyring::Entry::new(KEYCHAIN_SERVICE, KEYCHAIN_ACCOUNT)
        .map_err(|e| format!("Keychain access failed: {}", e))?;

    match entry.get_password() {
        Ok(password) => {
            // password 存的就是 base64 的 32 字节 AES key
            Ok(KeychainResult { found: true, key_b64: Some(password) })
        }
        Err(keyring::Error::NoEntry) => {
            // 首次运行：生成新 key 并写入 keychain
            let mut key = [0u8; 32];
            use rand::RngCore;
            rand::thread_rng().fill_bytes(&mut key);
            let key_b64 = base64::Engine::encode(&base64::engine::general_purpose::STANDARD, &key);
            entry.set_password(&key_b64)
                .map_err(|e| format!("Keychain write failed: {}", e))?;
            Ok(KeychainResult { found: true, key_b64: Some(key_b64) })
        }
        Err(e) => Err(format!("Keychain read failed: {}", e)),
    }
}

#[derive(Debug, Serialize, Deserialize)]
pub struct PlatformInfo {
    pub os: String,
    pub arch: String,
    pub hostname: String,
}

#[tauri::command]
pub async fn get_platform_info() -> Result<PlatformInfo, String> {
    Ok(PlatformInfo {
        os: std::env::consts::OS.to_string(),
        arch: std::env::consts::ARCH.to_string(),
        hostname: hostname::get()
            .map_err(|e| format!("hostname failed: {}", e))?
            .to_string_lossy()
            .to_string(),
    })
}
```

### 🔴 P0-2：错误信息泄漏 API Key 前缀

**严重度**：HIGH（信息泄漏 + 信任崩塌）
**风险描述**：
- Anthropic 401 错误返回 `"Invalid API key: sk-ant-api03-xxx..."`（含明文前缀）
- OpenAI 401 返回 `"Incorrect API key provided: sk-proj-xxx..."`
- Google 403 返回 `"API key not valid. Key starts with: AIza..."`
- 如果 v0.2 把这些错误直接返回给前端 → 用户截图发网上 → 前缀泄漏

**当前防护**：
- ❌ `response.ts:handlePrismaError` 直接 `fail(c, 500, message)` 把原始 message 返回
- ❌ `index.ts:onError` 也是 `return fail(c, 500, message)` 原始 message 返回
- ⚠️ 错误清洗**完全没做**

**攻击场景**：
1. 用户 A 用自己密钥 `sk-ant-api03-AAA...`
2. 密钥过期后调 LLM，401 错误信息含完整 `sk-ant-api03-AAA...`
3. Cosmgrid-Agent UI 直接显示在 alert 框里
4. 用户截图发 issue / Twitter / B 站 → 前缀公开

**改进建议（必须改）**：

```typescript
// app/src/lib/error-sanitizer.ts（新文件）

/**
 * API Key 前缀正则（覆盖主流 provider）
 * 设计原则：只匹配"前缀"（前 8-15 字符），不匹配完整 key
 */
const API_KEY_PATTERNS = [
  // Anthropic: sk-ant-api03-xxxxx 或 sk-ant-xxxxx
  /sk-(?:ant-)?(?:api03-)?[A-Za-z0-9_\-]{8,}/g,
  // OpenAI: sk-proj-xxxxx 或 sk-xxxxx
  /sk-(?:proj-)?[A-Za-z0-9_\-]{20,}/g,
  // Google: AIzaSyAxxxxx (39 字符)
  /AIza[A-Za-z0-9_\-]{30,}/g,
  // DeepSeek: sk-xxxxx (35 字符)
  /sk-[a-f0-9]{32,}/g,
  // 通用 Bearer token
  /Bearer\s+[A-Za-z0-9_\-\.]{20,}/gi,
];

const PROVIDER_NAMES = [
  "anthropic", "openai", "google", "deepseek", "qwen", "glm",
  "moonshot", "zhipu", "baichuan", "mistral", "groq", "xai",
];

/**
 * 把错误信息里可能包含的 API Key 前缀/敏感信息清掉
 * 保留错误结构（让用户能排查"是什么类型的错"），移除具体凭证值
 */
export function sanitizeError(error: unknown): string {
  let message = error instanceof Error ? error.message : String(error);

  // 1. 移除 API Key（保留前缀 3-4 个字符供识别）
  for (const pattern of API_KEY_PATTERNS) {
    message = message.replace(pattern, (match) => {
      const prefix = match.slice(0, 7); // sk-ant- 或 AIzaSyA
      return `${prefix}***[已隐藏]***`;
    });
  }

  // 2. 移除 Authorization header 值
  message = message.replace(
    /(?:Authorization|x-api-key)["']?\s*[:=]\s*["']?[A-Za-z0-9_\-\.]{20,}/gi,
    "$1: ***[已隐藏]***"
  );

  // 3. 移除 base64 编码的 key（长 base64 串）
  message = message.replace(/\b[A-Za-z0-9+/]{40,}={0,2}\b/g, "***[base64 隐藏]***");

  // 4. 截断超长错误（防止堆栈泄漏）
  if (message.length > 500) {
    message = message.slice(0, 500) + "...[已截断]";
  }

  return message;
}

/**
 * 给 LLM 提供商特定错误分类（用户友好的中文文案）
 */
export function classifyLlmError(error: unknown): {
  code: "auth" | "rate_limit" | "quota" | "network" | "model_not_found" | "unknown";
  userMessage: string;
} {
  const msg = error instanceof Error ? error.message : String(error);
  const lower = msg.toLowerCase();

  if (lower.includes("401") || lower.includes("unauthorized") || lower.includes("invalid api key") || lower.includes("authentication")) {
    return {
      code: "auth",
      userMessage: "API Key 无效或已过期，请到供应商页面检查后重新填写",
    };
  }
  if (lower.includes("403") || lower.includes("forbidden")) {
    return {
      code: "auth",
      userMessage: "API Key 无权限访问该模型，请检查是否启用了对应模型权限",
    };
  }
  if (lower.includes("429") || lower.includes("rate limit") || lower.includes("too many requests")) {
    return {
      code: "rate_limit",
      userMessage: "请求频率超限，请稍后再试（建议 1 分钟后再试）",
    };
  }
  if (lower.includes("quota") || lower.includes("insufficient") || lower.includes("balance")) {
    return {
      code: "quota",
      userMessage: "账户余额不足，请到供应商充值后再试",
    };
  }
  if (lower.includes("model not found") || lower.includes("404")) {
    return {
      code: "model_not_found",
      userMessage: "模型名称不存在，请检查 Model ID 是否正确（区分大小写）",
    };
  }
  if (lower.includes("network") || lower.includes("econnrefused") || lower.includes("timeout")) {
    return {
      code: "network",
      userMessage: "网络连接失败，请检查 baseUrl 是否可访问",
    };
  }
  return {
    code: "unknown",
    userMessage: `调用失败：${sanitizeError(error)}`,
  };
}
```

#### 服务端限流（`app/server/middleware/rate-limit.ts`，新文件）

```typescript
// 简单的内存限流（单机够用，不需要 Redis）
// 生产环境建议用 better-sqlite3 / lru-cache 持久化

import type { Context, Next } from "hono";

interface RateLimitConfig {
  windowMs: number;       // 时间窗口（毫秒）
  maxRequests: number;    // 窗口内最大请求数
}

interface Bucket {
  count: number;
  resetAt: number;
}

const buckets = new Map<string, Bucket>();

export function rateLimit(config: RateLimitConfig) {
  return async (c: Context, next: Next) => {
    // 桌面应用：通常 127.0.0.1，per-IP 等于 per-user
    // 但 Tauri 多窗口可能不同 IP，统一用 per-user (从 API key hash 推)
    const identifier = c.req.header("x-forwarded-for") ?? "127.0.0.1";
    const now = Date.now();

    const bucket = buckets.get(identifier);
    if (!bucket || bucket.resetAt < now) {
      buckets.set(identifier, { count: 1, resetAt: now + config.windowMs });
    } else if (bucket.count >= config.maxRequests) {
      const retryAfter = Math.ceil((bucket.resetAt - now) / 1000);
      c.header("Retry-After", String(retryAfter));
      return c.json(
        { success: false, error: `请求过于频繁，${retryAfter} 秒后再试` },
        429
      );
    } else {
      bucket.count++;
    }

    await next();
  };
}

// 定期清理过期 bucket（防内存泄漏）
setInterval(() => {
  const now = Date.now();
  for (const [key, bucket] of buckets) {
    if (bucket.resetAt < now) buckets.delete(key);
  }
}, 60_000);
```

## 三、加密方案总评（v0.2 → v0.3 → v0.4 演进路径）

| 维度 | v0.2 草案（固定密钥派生） | v0.2 推荐（OS keychain + 派生 fallback） | v0.3 演进（用户密码派生） | v0.4 演进（硬件加密） |

### v0.2 安全方案 = 9 条规则

1. **加密**：OS keychain 优先（macOS Keychain / Windows DPAPI / Linux Secret Service） + PBKDF2 派生 fallback（600k 次迭代 + 设备指纹 salt）

3. ⚠️ Tauri capabilities 加注释（哪些命令禁止 allow 的清单）

## 九、给不懂技术的用户的"通俗版"风险总结

> **用户（小白）看这里就够了**：
>
> 你们最担心的"API Key 被人偷走刷我余额"，v0.2 草案**有风险但可控**。
>
> 1. **存 API Key**：v0.2 会用 AES-256 加密，存在你电脑的"钥匙串"（macOS Keychain / Windows 凭据管理器）里。**就算有人偷走你电脑上的数据库文件，也解不开**——除非他同时拿到你电脑登录密码。
>
> 2. **输错了 API Key**：界面会告诉你"Key 无效"，但**不会**显示完整 Key 内容（即使 Key 本身泄漏了，截图发网上也安全）。
>
> 3. **测试连接点太多次**：客户端会强制等 2 秒才能再点（防误触）；服务端会拦截超过 5 次/分钟的请求（防恶意刷）。
>
> 4. **App 自身安全**：网页代码（HTML/CSS/JS）会限制只能从本地加载，不能从网络加载乱七八糟的脚本（防网页钓鱼）。
>
> **一句话总结**：v0.2 上线前，5 个核心安全项必须通过（详见"最终推荐"5 条），其他都是锦上添花。
